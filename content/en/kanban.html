What is Kanban?

Kanban is a system for visualizing work, limiting work in progress (WIP), and optimizing flow. It helps teams manage tasks efficiently by making the status of work visible and encouraging continuous improvement.

Core Principles

Visualize Work

Use a Kanban board with columns representing stages of the workflow (e.g., To Do → In Progress → Done). Tasks are represented by cards that move across columns.

Limit Work in Progress (WIP)

Set limits on how many tasks can be in each stage at once. This prevents overload and ensures focus.

Manage Flow

Track how tasks move through the process and identify bottlenecks to improve efficiency.

Make Process Policies Explicit

Define clear rules for how work moves between stages.

Improve Collaboratively, Evolve Experimentally

Use metrics and feedback to continuously improve the process.

Kanban Board Example

Backlog To Do In Progress Review Done
  • Collect requirements
  • Define acceptance criteria
  • Draft architecture notes
  • Create user story #101
  • Prepare sprint plan outline
  • Implement login form
  • Write unit tests for auth
  • Code review: API endpoints
  • Validate test coverage
  • Project setup
  • CI pipeline skeleton

Working Agreements (Policies)

  1. Limit Work In Progress in In Progress to 3 items.
  2. Only pull new work when capacity is available.
  3. Each card must include a clear definition of done.
